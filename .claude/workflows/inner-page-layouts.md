# Inner Page Layout Variations

Don't use the same inner page layout on every build. Check the last 2 builds' inner pages and vary the structure.

## Services Page Layouts

### A. Zigzag (alternating image/text)
Rows alternate: image-left/text-right, then image-right/text-left.
Best for: 4-6 services with strong imagery.

### B. Feature Cards + Detail Accordion
Hero grid of service cards (bento layout), each expandable with full details.
Best for: Many services (6+), mobile-friendly.

### C. Sidebar Navigation
Fixed left sidebar with service list. Clicking/scrolling reveals the service detail on the right.
Best for: Premium businesses, lots of detail per service.

### D. Timeline/Journey
Vertical timeline showing services in the order a typical client experiences them.
Best for: Process-heavy trades (renovation, construction).

### E. Category Tabs
Tabbed interface grouping services into categories.
Best for: Businesses with distinct service categories.

## About Page Layouts

### A. Story + Image (standard)
Left column text, right column image, stats grid below.

### B. Timeline Biography
Vertical timeline of business milestones with images per milestone.
Best for: Businesses with a long history.

### C. Split Hero + Values Grid
Full-width hero image with overlay text, then a values/principles grid below.
Best for: Owner-story-driven businesses.

### D. Magazine Layout
Large feature image, pull quotes, multi-column text with drop caps.
Best for: Premium/artisan businesses.

## Contact Page Layouts

### A. Form + Sidebar (standard)
Form on left, contact info + map on right.

### B. Map Hero + Form Below
Full-width map at top, form section below with contact cards.
Best for: Location-critical businesses.

### C. Minimal Form
Clean single-column centred form with no sidebar. Map in separate section below.
Best for: Minimalist archetypes.

### D. Multi-Step Form
Step 1: What service? Step 2: Project details. Step 3: Contact info.
Best for: Complex project-based businesses.

## Rules

1. **No repeat within 2 builds.** Check what the last 2 builds used.
2. **Match the archetype.** Premium archetypes get premium layouts (C/D). Practical archetypes get practical layouts (A/B).
3. **Record in the plan.** Each inner page task must specify which layout variant to use.
4. **Don't default to A.** If unsure, pick B or C — they force more creative thinking.
